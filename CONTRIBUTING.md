# Contributing

Thank you for contributing to AethOS.

## Code of conduct

Be respectful, constructive, and assume good intent.

## How to contribute

1. Fork the repository (or use a branch if you have write access).
2. Create a focused feature branch.
3. Make changes with tests where practical.
4. Run **`pytest`** and **`python -m compileall -q app`** before opening a PR.
5. Open a **Pull Request** with a clear description.

## Development setup

See [development/setup.md](setup.md).

## Style

- Match existing patterns (types, naming, logging).
- **Ruff:** `ruff check app tests` (see `pyproject.toml`).
