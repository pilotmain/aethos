# Security

## Reporting a vulnerability

If you believe you have found a security issue in this repository, please report it privately rather than opening a public issue.

**Preferred:** contact the repository maintainers through the security advisory flow on GitHub (**Security → Report a vulnerability**), if enabled for this repo.

If that is not available, email the maintainer contact listed in the repository README or organization profile.

## What to include

- A short description of the issue and its impact  
- Steps to reproduce (or a proof-of-concept), if possible  
- Affected versions or commit range, if known  

Please avoid posting exploit details in public issues or discussions until maintainers have had a reasonable chance to respond.

## Scope

Reports should relate to **this codebase** (API, worker, tooling shipped in-repo). Hosted services, third-party integrations, or unrelated infrastructure may be out of scope unless explicitly covered by the maintainers.

## Disclosure

Maintainers aim to acknowledge valid reports in a timely manner and coordinate disclosure after a fix or mitigation is available. Exact timelines depend on severity and complexity.
