# Adaptive runtime intelligence

Phase 4 Step 1 adds **advisory-first** operational optimization to runtime truth:

- `adaptive_runtime_intelligence`
- `adaptive_operational_learning`
- `operational_optimization_signals`
- `runtime_adaptation_history` (bounded in `~/.aethos/aethos.json`)

All signals are explainable, runtime-visible, and require operator review before any automated action.

**API:** `GET /api/v1/mission-control/runtime/evolution`  
**CLI:** `aethos runtime evolution`

Phase 4 Step 2 adds **adaptive coordination** (`adaptive_coordination`, `coordination_quality`, `runtime_balancing`, `adaptive_execution_signals`) — still advisory-first.
