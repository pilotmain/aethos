# Configuration

Environment variables are loaded from **`.env`** at the repository root (see **`.env.example`** for the full, commented list). Do not commit real secrets.

## Common variables

| Variable | Description | Typical |
| --- | --- | --- |
| `DATABASE_URL` | SQLAlchemy URL | SQLite default in `.env.example`; Postgres in Docker |
| `NEXA_SECRET_KEY` | Signing / crypto for web and stored keys | **Required** on shared hosts |
| `NEXA_WEB_API_TOKEN` | Bearer for Mission Control when set | Generated by setup wizard |
| `TELEGRAM_BOT_TOKEN` | Telegram bot | Optional until you run the bot |
| `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` | LLM providers | Required when `USE_REAL_LLM=true` |
| `USE_REAL_LLM` | Use real models vs local fallbacks | `false` for quick smoke |
| `NEXA_WORKSPACE_ROOT` | Host / sandbox workspace | e.g. `$HOME/aethos_workspace` |
| `NEXA_SANDBOX_EXECUTION_ENABLED` | Allow sandbox NL + executor | `false` by default |

## License / Pro

| Variable | Description |
| --- | --- |
| `NEXA_LICENSE_KEY` | Signed license token (when using commercial verifier) |
| `NEXA_LICENSE_PUBLIC_KEY_PEM` | Public key for verifying tokens |
| `AETHOS_PRO_ENABLED` | Opt-in for Pro import paths when package + license align |

See [open-core.md](open-core.md) and [OPEN_CORE_EXTENSIONS.md](OPEN_CORE_EXTENSIONS.md).

## Ports

- Default API in many docs: **8010** (align `API_BASE_URL` / `NEXT_PUBLIC_*` with your run).
- Web dev server: often **3000** — see [WEB_UI.md](WEB_UI.md).

Always treat **`.env.example`** as the source of truth for names and safe defaults.
